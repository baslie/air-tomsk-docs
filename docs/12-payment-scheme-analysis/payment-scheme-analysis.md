## 3. Проблема схемы 30/70

При схеме **30/70** подрядчик получает предоплату **87 000 ₽**. Подписки оплачиваются помесячно:

- **Месяц 1:** −23 119 ₽ → на работу остаётся **63 881 ₽**
- **Месяц 2:** ещё −23 119 ₽ → на работу остаётся **40 761 ₽**

Итого за 1,5 месяца работы из предоплаты на руках — лишь **40 761 ₽** (**14%** от стоимости проекта). Фактически подрядчик **1,5 месяца работает за 14%** от суммы, неся все затраты на инструменты за свой счёт. Оставшиеся **70% (203 000 ₽)** он получает только после сдачи проекта.

---

## 4. Предлагаемая схема

Исполнитель соглашается на схему **30/70**, но стоимость подписок выносится за рамки суммы проекта. Клиент доплачивает за подписки отдельно, ежемесячно.

| Статья | Сумма |
|--------|------:|
| Стоимость работы | 290 000 ₽ |
| Предоплата 30% | 87 000 ₽ |
| Финальный платёж 70% | 203 000 ₽ |
| Подписки (~23 119 ₽/мес × 2 мес) | 46 239 ₽ (отдельно) |
| **Итого для клиента** | **336 239 ₽** |

**Выгода для исполнителя:** предоплата 87 000 ₽ целиком идёт на работу — не нужно покрывать подписки из своего кармана.

**Выгода для клиента:** сохраняется желаемая схема 30/70, а расходы на подписки прозрачны и оплачиваются по факту использования.
