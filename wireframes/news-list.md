## Описание элементов

### [FILTERS] — Фильтры (опционально)
| Фильтр | Тип | Значения |
|--------|-----|----------|
| Год | select | 2025, 2024, 2023, ... |
| Месяц | select | Все, Январь, Февраль, ... |

**Поведение:**
- При изменении фильтра — перезагрузка с query-параметрами
- URL: `/news/?year=2025&month=01`

### [NEWS-CARD] — Карточка новости
| Элемент | Тип | Описание |
|---------|-----|----------|
| Изображение | img | Превью новости (WebP, lazy loading) |
| Дата | time | Дата публикации (datetime атрибут) |
| Заголовок | h3 | Заголовок новости (до 100 символов) |
| Анонс | p | Краткий текст (до 150 символов) |
| Кнопка | a | Ссылка на детальную страницу |

**Структура карточки:**
```html
<article class="news-card">
  <a href="/news/{slug}/">
    <img src="..." alt="..." loading="lazy">
  </a>
  <time datetime="2025-01-15">15 января 2025</time>
  <h3><a href="/news/{slug}/">Заголовок новости</a></h3>
  <p>Краткий анонс новости...</p>
  <a href="/news/{slug}/" class="btn">Подробнее</a>
</article>
```

### [NEWS-GRID] — Сетка новостей
| Параметр | Значение |
|----------|----------|
| Количество на странице | 6 (настраивается) |
| Колонки (desktop) | 3 |
| Колонки (tablet) | 2 |
| Колонки (mobile) | 1 |
| Отступы | gap: 24px |

### [PAGINATION] — Пагинация
| Элемент | Описание |
|---------|----------|
| Стрелки | ← → для перехода |
| Номера | Номера страниц (текущая выделена) |
| Многоточие | ... для пропуска страниц |
| Счётчик | "Показано X-Y из Z" |

**URL-структура:**
- `/news/` — первая страница
- `/news/?page=2` — вторая страница
- `/news/?year=2024&page=3` — с фильтрами

## Адаптивное поведение

### Desktop (≥1280px)
- Вертикальное меню слева (фиксированное)
- Сетка новостей: 3 колонки
- Фильтры в одну строку
- Пагинация по центру

### Tablet (768px — 1279px)
- Меню в "бургер"
- Сетка новостей: 2 колонки
- Фильтры: год и месяц в ряд
- Контент на всю ширину

### Mobile (≤767px)
- "Бургер-меню"
- Сетка новостей: 1 колонка
- Фильтры: вертикально
- Карточки на всю ширину
- Упрощённая пагинация (только стрелки)

## Технические заметки

1. **Производительность:**
   - Lazy loading для изображений
   - Серверная пагинация
   - Кэширование списка

2. **Изображения:**
   - Размер превью: 400x250px
   - Формат: WebP с fallback на JPEG
   - Srcset для разных разрешений

3. **Админка:**
   - Управление через Django Admin
   - Поля: заголовок, slug, дата, анонс, текст, изображение
   - Статус публикации (черновик/опубликовано)
